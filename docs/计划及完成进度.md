## 一、问题诊断：为什么 TSNet 仿真结果"一直不好"

### 1.1 根本原因

TSNet 是为**市政供水管网**设计的水击求解器（基于 EPANET），将其用于**石油压裂井筒-裂缝系统**存在多方面的物理模型不匹配，调参无法解决。

### 1.2 关键物理差距清单（按对结果影响排序）

| # | 差距 | TSNet 现状 | 压裂实际 | 影响 |
|---|---|---|---|---|
| 1 | **裂缝物理模型** | 盲端圆管 + 末端 emitter \(Q=k\sqrt H\) | 平面裂缝，壁面柔度主导波速，分布滤失 | 反射幅值/相位/频率响应全错 |
| 2 | **泵/井口边界** | 离心泵三点特性曲线 \(h_p=aQ^2+bQ+c\) | 柱塞泵停泵 = 流速阶跃 V→0 | 初始 Joukowsky 跳变符号/幅值不对 |
| 3 | **流体本构** | 牛顿水，\(\nu=1.004\times10^{-6}\) 写死 | 滑溜水/胶液/交联液 + 支撑剂，非牛顿 | 摩阻/波速/密度不准 |
| 4 | **初值求解器** | EPANET DD + Hazen-Williams + LPS 单位 | 压裂稳态需自建（井口流量驱动） | 初值不自洽，t=0 产生伪压力波 |
| 5 | **离散化精度** | 全网波速协调，1.2m 段在 1450 m/s 下 N≈0.83 → round 到 1，波速被改 20% | 单管精确 Courant | 深部反射到达时间误差大 |
| 6 | **摩阻模型** | 稳态达西 / Brunone（符号存疑 `# TO DO`） | 滑溜水近似牛顿，达西可接受 |Brunone 不可全信 |

### 1.3 已踩过的坑（项目日记 2026-04-21）

- `simulator.py:443-458` 已对"趾端死端初值"打过 Joukowsky 补丁（`H_toe = H_up + a/g·V`，`V_toe=0`），但这只是补丁，多缝/zone/dense 拓扑下需反复改
- EPANET 稳态水头（有 demand）与 MOC 死端边界不兼容，存在 ~a/g·V ≈ 138m 的水头缺口

### 3.2 物理模型与控制方程

#### 井筒（1D MOC，滑溜水）

\[
\frac{\partial H}{\partial t}+\frac{a^2}{g}\frac{\partial V}{\partial x}=0,\qquad
\frac{\partial V}{\partial t}+g\frac{\partial H}{\partial x}+\frac{f}{2D}V|V|+g\theta=0
\]

参数（滑溜水默认值，全部从 config 读取）：
- ρ=1000 kg/m³，ν=1.0×10⁻⁶ m²/s（近似牛顿）
- a=1450 m/s（钢套管 + 滑溜水）
- f：稳态达西，按 Re + Colebrook-White 显式近似
- θ=0（水平井）

#### 特征线（C⁺/C⁻）

\[
C^+: V_P + \frac{g}{a}H_P = V_1 + \frac{g}{a}H_1 - J_1 + \frac{g}{a}\Delta t\,V_1\theta
\]
\[
C^-: V_P - \frac{g}{a}H_P = V_2 - \frac{g}{a}H_2 - J_2 - \frac{g}{a}\Delta t\,V_2\theta
\]

内节点：\(H_P=(C^+ + C^-)/(2g/a)\)，\(V_P=C^+ - (g/a)H_P\)

#### 井口边界（停泵，流速阶跃）

\[
t< t_s: V_P=V_0;\qquad t\ge t_s: V_P=0
\]
\[
H_P=(V_P + C^-)/(g/a)
\]

#### 趾端边界（两种模式）

- **dead_end**（实际井筒用）：\(V_P=0\)，\(H_P=C^+/（g/a)\)，反射系数 +1
- **reservoir**（Joukowsky 验证用）：\(H_P=H_{\text{toe}}\)，\(V_P=C^+ - (g/a)H_P\)，反射系数 -1

#### 裂缝节点

\[
Q_f(t) = C_f\,\dot H(t) + k_{\text{leak}}\sqrt{H(t)-H_{\text{ext}}}
\]

- 节点连续性 + 半隐式离散 → 关于 \(H_P^n\) 的非线性方程 → Newton 迭代
- **2026-07-03 边界修复**：`H_old` 改用缝节点两侧邻居的空间平均  
  `H_old_avg = 0.5·(H[i_f-1] + H[i_f+1])`，消除奇偶网格解耦；恢复标准 dt 后向欧拉，删除 2Δt 跨步与 Robert-Asselin 滤波
- `H_ext`：地层孔隙压力水头 [m]，仅影响滤失项 \(k_{\text{leak}}\sqrt{H-H_{\text{ext}}}\)

#### 时空离散

- 单一井筒，\(N=L/(a\,\Delta t)\) 取整，dt 微调使 \(a\Delta t/\Delta x=1\)（Courant=1，无数值耗散）
- 缝位置对齐到最近网格点（误差 ≤Δx/2）
- 含沿程摩擦梯度的自洽稳态初始化：\(H(x)=H_{\text{toe}}+(f/D)(V_0|V_0|/2g)(L-x)\)

### 4.1 `wellbore_moc.py`（~700 行，含 Step 3 裂缝边界 + Brunone 非定常摩阻）

**核心组件**：

1. `MocConfig` 数据类：所有物理参数显式化（几何/流体/摩阻/时间/边界/趾端）
   - 派生量自动计算：area, N, dx, dt_adj, a_adj, n_steps
   - `toe_bc` 支持 `dead_end` / `reservoir` 两种模式
   - `wellhead_bc` 支持 `velocity_step` / `ramp` 两种模式
   - `friction_model` 支持 `steady` / `quasi-steady` / `brunone`

2. 摩阻函数：
   - `reynolds(V, D, nu)` — 雷诺数
   - `darcy_friction_factor(Re, K_D, model)` — 层流 64/Re + 紊流 Zigrand-Swamee 显式近似
   - `friction_term_J(f, D, V, dt)` — 稳态达西摩阻项 \(J=f\Delta t V|V|/(2D)\)
   - **`brunone_k(Re)` / `brunone_k_vec(Re_arr)`**（Step 3b 新增）— Brunone 系数 k=√C/2，标量+向量化版本
   - **`brunone_friction_Ju(k, dt, dVdt, dVdx, V, a)`**（Step 3b 新增）— 非定常摩阻项 \(J_u=\frac{k}{2}\Delta t(\dot V + a\,\text{sign}(V)|V'|)\)

3. **`solve_fracture_node()`**（Step 3 新增；2026-07-03 改用邻居空间平均 `H_old_avg`）— 裂缝节点非线性方程求解器
   - 集总柔度 + 分布滤失边界：\(Q_f = C_f\dot H + k_{\text{leak}}\sqrt{H-H_{\text{ext}}}\)
   - 半隐式离散 → \(\alpha H_P + \beta\sqrt{H_P-H_{\text{ext}}}+\gamma=0\)
   - Newton 迭代（k_leak=0 时退化为线性，直接求解）
   - 返回 H_P, V_left, V_right, Q_f

4. `simulate_wellbore(cfg, ...)` — 主仿真器
   - 自洽稳态初始化（含沿程摩擦梯度）
   - **双流速数组** `V_prev_left` / `V_prev_right`：非缝节点 left=right=V；缝节点 left/right 分离，确保 C⁺/C⁻ 正确传播
   - 时间推进：内节点 C⁺/C⁻ 联立（向量化）→ 裂缝节点覆盖 → 井口边界 → 趾端边界
   - 缝位置自动对齐到最近网格点，检测重合并报错
   - 返回完整时空场 + 缝节点水头/流量时程

5. `simulate_case(config, friction_model)` — dataset_builder 调用入口
   - 从 config.json 构造 MocConfig + 裂缝参数
   - 返回与 TSNet 路径兼容的 dict schema（junction_heads 已填充）

**关键技术点**：
- Courant 精确：N=round(L/(a·dt))，a_adj=dx/dt，确保 a·dt/dx=1（无数值耗散）
- 摩擦梯度自洽初值：H(x)=H_toe + (f/D)(V0|V0|/2g)(L-x)，消除 EPANET 式初值不兼容
- 死端 + Joukowsky 一致：H_toe = H_{N-1} + (a/g)·V_{N-1}
- **双流速数组**解决缝节点速度不连续问题：V_left 供上游邻居 C⁻，V_right 供下游邻居 C⁺

### 4.4 关键物理发现

#### 发现 1：压裂停泵水击是"源头截断型" Joukowsky（Step 2）

**压裂停泵水击初始压力变化为 -aV0/g（下降），不是 +aV0/g（上升）。**

- 流体从井口流向趾端（+x），停泵封堵井口 → 流体继续向 +x 流走 → 井口产生低压区
- 经典 +aV0/g 是"下游阀门关闭"（流体撞向阀门）
- TSNet 把泵当离心泵（reservoir=0 + 泵曲线），停泵时泵出口水头通过特性曲线漂移，**给不出正确的初始 Joukowsky 跳变**——这是 TSNet 结果不对的关键原因之一
- 现场压裂 PT 数据看到的"正水击尖峰"其实是 **2L/a 后从趾端/裂缝反射回来的反弹波**，不是停泵瞬时的初始变化
- 反射序列：[ts, ts+2L/a) H0-ΔH → [ts+2L/a, ts+4L/a) H0+ΔH → 振荡衰减

#### 发现 2：井口闭端反射加倍效应（Step 3）

**缝反射波到达井口后，因井口 V=0（闭端，反射系数 +1），反射波幅值加倍为 2ΔH。**

- 缝反射波本身幅值 = +ΔH·exp(-t/τ)（Laplace 域 Γ(s)=-sCfZw/(2+sCfZw) 对阶跃的响应）
- 到达井口闭端后反射系数 +1，叠加 → 井口观测到的总跳变 = **2ΔH**
- 井口水头：\(H(t) = (H_0-\Delta H) + 2\Delta H \cdot e^{-(t-t_{\text{arrive}})/\tau}\)
- 衰减时间常数 τ = Cf·Zw/2 = Cf·a/(2gA)，与闭端加倍无关
- **实测**：Cf=1e-4 m² → τ_ana=0.482s, τ_fit=0.478s（R²=0.99996），幅值 285.6m vs 解析 295.5m（3.3% 摩擦误差）

#### 发现 3：裂缝频率相关反射特性（Step 3）

集总柔度裂缝的反射系数（Laplace 域）：
\[
\Gamma(s) = \frac{-s\,C_f\,Z_w}{2 + s\,C_f\,Z_w},\qquad Z_w=\frac{a}{gA}
\]

- 高频（s→∞）：Γ→-1，裂缝像开端（压力波→抽吸波），强反射
- 低频（s→0）：Γ→0，裂缝透明，无反射
- 时域对阶跃的响应：+ΔH·exp(-t/τ)，指数衰减
- 与真实裂缝"低频透射、高频反射"物理一致——这是 TSNet 盲管模型做不到的

#### Cf 取值范围修正

原计划估计 Cf∈[0.001, 0.1] m²，验证后发现：
\[
\tau = \frac{C_f \cdot Z_w}{2},\quad Z_w = \frac{a}{gA} \approx 9638\;\text{s/m}^2
\]
- Cf=0.001 → τ≈4.8s（衰减太慢，仿真窗口内看不到）
- Cf=0.0001 → τ≈0.48s（合理，可见衰减）
- Cf=0.00001 → τ≈0.048s（衰减太快，接近瞬时）

**修正后推荐 Cf 采样范围：[1×10⁻⁵, 1×10⁻³] m²**（对应 τ≈0.05–4.8s）

| 5. 无缝退化 | <1×10⁻¹⁰ m | **3.9×10⁻¹¹ m** | PASS |

**关键物理发现**：

1. **Brunone 非定常摩阻正确实现衰减**：衰减比 0.47（Brunone） vs 1.73（稳态），Brunone 使振荡能量在 100s 内衰减到早期的 47%，而稳态摩阻因波形叠加 RMS 反而"增长"到 173%

2. **稳态摩阻 RMS "增长"不是能量增长**：稳态衰减比 1.73 > 1 是因为多反射源叠加使波形复杂化（相位关系重组），RMS 增大是波形形态变化，不是物理能量增长。peak-to-peak 指标同样受此影响，RMS 是更好的能量指标。

3. **反射周期匹配现场**：2L/a = 6.9s ≈ 现场 7s（L=5000m, a=1450m/s），验证了现场波形 ~7s 周期来自趾端反射

4. **向量化优化**：`brunone_k_vec` 替代 Python 循环，100000 步 × 3448 节点仿真从 >20 分钟降至 ~38 秒（50x 加速）

5. **内存优化**：`store_full_field=False` + `snapshot_times` 机制，100s 仿真内存从 ~5.5GB 降至 ~10MB

**输出文件**：
- `output/brunone_validation.png`（2×2 验证图）
- `output/brunone_validation.json`（结果 JSON）

### 4.6 `validate_moc_dual_fracture.py` — 双缝验证（Step 4a，2026-07-02）

**验证场景**：L=5000m, x_f=[4500, 4800]m, 相同 Cf=1e-4 m², k_leak=0, Brunone, tf=100s

**验证结果**（5/5 PASS）：

| 判定 | 实测 | 结论 |
|---|---|---|
| 1. Joukowsky 跳变 | 0.0000% | PASS |
| 2. 双缝+趾端反射（差信号验证） | 缝1 +196m, 缝2 -30m(叠加衰减), 趾端 +223m | PASS |
| 3. 振荡衰减(RMS) | Brunone 0.43 vs 稳态 0.53, 末期RMS比 0.79 | PASS |
| 4. 长期稳定性 | 48.5m < 73.9m, 无NaN | PASS |
| 5. 无缝退化 | 4.3e-11 m | PASS |

**关键物理发现**：

1. **双缝反射独立可分辨**：缝1(7.21s)、缝2(7.62s)、趾端(7.90s)三个反射间距 0.41s 和 0.28s，在差信号中清晰可见

2. **差信号验证法**：直接验证 H 的 step 会因三反射叠加而误判（缝2的净 step 为负）。用 `H_withFrac - H_noFrac` 差信号隔离缝贡献后，缝1 step=+196m（正向），缝2 step=-30m（负向，因叠加在缝1指数衰减上），两者 |step| 均远大于噪声

3. **缝2负向 step 的物理**：缝2反射到达时（7.62s），缝1反射已衰减 0.41s（exp(-0.41/0.48)≈0.42），缝1衰减速率 > 缝2上升速率 → 差信号净下降。这不是物理错误，而是多反射叠加的正常行为

4. **Brunone 双缝衰减比 0.43 < 稳态 0.53**：双缝场景下 Brunone 衰减仍强于稳态，末期 RMS 比 0.79 < 0.9

### 4.8 `cepstrum_mocdata.py` — 井口水头倒谱分析（Step 4c，2026-07-03）

**功能**：对 MOC 井口水头时序做倒谱分析，供 `validate_moc_test_b.py` 等脚本可视化调用。

**处理流水线**：

1. **预处理** `preprocess_moc_head`：三次样条重采样 → 停泵后截取 (t≥ts) → 去均值
2. **可选求导** `derivative=True`：np.gradient 1/2 阶，锐化反射边沿

6. **深度映射**：depth = q·v/2；响应量 response = -C

| 行 | 内容 |

### 4.9 裂缝边界空间平均修复（2026-07-03）

**问题**：原缝节点使用本节点历史 `H_prev[i_f]` 作 `H_old`，与 C⁺/C⁻ 特征线网格不一致，产生奇偶解耦与数值振荡。

**修复**（`wellbore_moc.py` 缝节点覆盖段）：

```python
H_old_avg = 0.5 * (H_prev[i_f - 1] + H_prev[i_f + 1])
```

- 删除 2Δt 跨步与 Robert-Asselin 滤波
- 恢复标准 dt 后向 Euler + Newton 求解

## 六、风险与注意事项

1. **Brunone 非定常摩阻符号存疑**（`solver.py:91` 自带 `# TO DO: check the sign`）— 当前自研核心先用稳态达西，滑溜水近似牛顿可接受；若后续需高保真衰减，需独立校核 Brunone 符号
2. **缝位置网格对齐误差** ≤Δx/2 — 对深部小间距缝（dense 模式 1.2m 间距）可能影响倒谱分辨率，需评估
3. **Cf 物理标定** — 集总柔度 \(C_f\) 的取值范围需从裂缝几何（半长/开度/模量）反推或现场标定；Step 3 验证后修正推荐范围为 [1×10⁻⁵, 1×10⁻³] m²（对应 τ≈0.05–4.8s，原 [0.001, 0.1] m² 衰减太慢）
4. **含滤失裂缝的稳态 V 分布** — 测试 B 已验证 k_leak>0 时 Q_f 稳态分量与解析一致；更一般稳态 V(x) 分布迭代求解仍待完善
5. **测试 B 平滑性 FAIL** — 抖动比 2.00 略超阈值，需排查是否为判定标准过严或反射叠加窗口选取问题
6. **保留 TSNet 对照** — 自研 MOC 与 TSNet 并行运行，可做 A/B 对比；最终若自研 MOC 全面胜出，再考虑移除 TSNet 调用
7. **双份代码维护** — `wellbore_moc_method/` 为归档主路径，`dataset_builder/` 为早期副本，避免分叉
